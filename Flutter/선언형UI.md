# 프로세스와 스레드 (Process & Thread)

## 1. 프로그램과 프로세스의 차이

프로그램은 단순히 실행 파일 형태로 존재하는 **정적인 데이터**이고,  
**프로세스(Process)** 는 그 프로그램이 메모리에 적재되어 실제로 실행되고 있는 **동적인 상태**를 의미한다.

운영체제는 실행 중인 각 프로세스에 독립된 메모리 공간을 할당한다.

### 프로세스의 메모리 구조

- **Code**: 실행할 프로그램의 코드(명령어) 저장
- **Data**: 전역 변수, 정적(static) 변수 저장
- **Heap**: 런타임에 동적으로 할당되는 메모리(객체 등)
- **Stack**: 함수 호출 시 생성되는 지역 변수, 매개변수 저장

즉, **프로세스는 운영체제로부터 자원을 할당받는 작업의 단위**이다.

---

## 2. 스레드(Thread): 프로세스 내의 실행 흐름

**스레드(Thread)** 는 프로세스 내에서 실제 작업을 수행하는 **실행의 최소 단위**이다.  
하나의 프로세스는 최소 하나 이상의 스레드(메인 스레드)를 가진다.

스레드의 가장 큰 특징은 **프로세스의 자원을 공유**한다는 점이다.

### 스레드의 메모리 공유 구조

- **공유하는 영역**: Code, Data, Heap
- **개별적인 영역**: Stack, Register

스택과 레지스터를 개별적으로 가지기 때문에 각 스레드는 독립적인 함수 호출과 실행 흐름을 가질 수 있다.

이처럼 자원을 공유하기 때문에 스레드 간 통신은 빠르고 효율적이지만,  
공유 데이터에 동시에 접근할 때 발생하는 문제를 적절히 관리해야 한다.

# 컨텍스트 스위칭 (Context Switching)

CPU는 한 번에 하나의 작업만 수행할 수 있다.  
하지만 매우 빠른 속도로 여러 작업을 번갈아 수행하기 때문에, 우리는 여러 프로그램이 동시에 실행되는 것처럼 느낀다.  
이 작업 교체 과정을 **컨텍스트 스위칭(Context Switching)** 이라고 한다.

## 1. 동작 방식

컨텍스트 스위칭은 다음과 같은 순서로 이루어진다.

1. 현재 실행 중인 프로세스 또는 스레드의 상태(Context)를 저장한다.
   - 프로세스는 **PCB(Process Control Block)**
   - 스레드는 **TCB(Thread Control Block)** 에 저장
2. 다음에 실행할 작업의 상태를 복구한다.
3. 복구된 상태를 CPU 레지스터에 적재하여 실행을 이어간다.

## 2. 프로세스 스위칭 vs 스레드 스위칭

### 프로세스 스위칭
- 서로 완전히 독립된 메모리 영역을 전환해야 한다.
- 캐시 메모리 초기화 등 추가 작업이 필요하다.
- 따라서 **오버헤드가 크다**.

### 스레드 스위칭
- 같은 프로세스 내에서 Code, Data, Heap을 공유한다.
- 저장하고 복구해야 할 정보가 상대적으로 적다.
- 따라서 **프로세스 스위칭보다 훨씬 빠르다**.

# 멀티 프로세스 vs 멀티 스레드

## 1. 멀티 프로세스 (Multi-process)

하나의 응용 프로그램을 여러 개의 프로세스로 구성하는 방식이다.  
예를 들어, 구글 크롬은 탭마다 별도의 프로세스로 동작할 수 있다.

### 장점
- 각 프로세스가 독립적이다.
- 하나의 프로세스가 종료되어도 다른 프로세스에 영향을 주지 않는다.
- 따라서 **안정성이 높다**.

### 단점
- 컨텍스트 스위칭 비용이 크다.
- 프로세스 간 데이터 공유를 위해 **IPC(Inter-Process Communication)** 같은 복잡한 통신 기법이 필요하다.

## 2. 멀티 스레드 (Multi-thread)

하나의 프로세스 내에서 여러 스레드를 생성하여 작업을 분담하는 방식이다.

### 장점
- 시스템 자원 소모가 적다.
- 스레드 간 통신이 매우 빠르다.
- 응답성이 좋아 실시간 서비스에 유리하다.

### 단점
- 자원을 공유하기 때문에 한 스레드의 오류가 전체 프로세스에 영향을 줄 수 있다.
- 데이터 동기화 처리가 까다롭다.

# 멀티 스레드 환경의 주요 이슈

멀티 스레드 프로그래밍에서는 다음 두 가지 핵심 문제를 해결해야 한다.

## 1. 동기화 (Synchronization)

여러 스레드가 동일한 공유 자원에 동시에 접근하여 데이터를 수정하면,  
데이터의 일관성이 깨질 수 있다.

이를 방지하기 위해 다음과 같은 잠금(Lock) 메커니즘을 사용한다.

- **뮤텍스(Mutex)**
- **세마포어(Semaphore)**

## 2. 교착 상태 (Deadlock)

두 개 이상의 스레드가 서로가 가진 자원을 기다리며  
무한히 대기하는 상태를 **교착 상태(Deadlock)** 라고 한다.

이 문제를 해결하려면 운영체제 수준 또는 애플리케이션 설계 단계에서  
예방하거나 회피하는 로직이 필요하다.
