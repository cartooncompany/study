# 상태 코드 분류

HTTP 상태 코드는 첫 번째 숫자에 따라 5가지 그룹으로 나뉩니다.

- **1xx**: 정보 제공
- **2xx**: 성공
- **3xx**: 리다이렉션
- **4xx**: 클라이언트 오류
- **5xx**: 서버 오류

---

## 🟢 1xx — 정보 제공

요청을 받았으며, 처리가 계속 진행 중임을 의미합니다.

실무에서 직접 자주 다루는 경우는 많지 않습니다.

| 코드 | 상태 | 의미 |
| --- | --- | --- |
| `100` | Continue | 요청의 일부가 정상적으로 수신되었으니, 나머지 데이터를 계속 전송하라는 의미 |
| `101` | Switching Protocols | 클라이언트가 요청한 프로토콜 전환(예: HTTP → WebSocket)을 서버가 승인함 |

---

## 🔵 2xx — 성공

요청이 성공적으로 처리되었음을 의미합니다.

| 코드 | 상태 | 의미 |
| --- | --- | --- |
| `200` | OK | 요청이 정상적으로 성공함 |
| `201` | Created | 요청이 성공했고, 새로운 리소스가 생성됨. 주로 `POST` 요청 후 사용 |
| `204` | No Content | 요청은 성공했지만, 응답 본문에 담을 데이터가 없음. 주로 `DELETE` 요청 후 사용 |

---

## 🟡 3xx — 리다이렉션

요청을 완료하려면 **추가 동작**이 필요함을 의미합니다.

주로 **페이지 이동**, **캐시 활용**과 관련이 있습니다.

| 코드 | 상태 | 의미 |
| --- | --- | --- |
| `301` | Moved Permanently | 요청한 리소스의 URI가 영구적으로 변경됨 |
| `302` | Found | 요청한 리소스의 위치가 일시적으로 변경됨 |
| `304` | Not Modified | 리소스가 수정되지 않았으므로, 브라우저가 기존 캐시를 사용해도 됨 |

---

## 🟠 4xx — 클라이언트 오류

클라이언트의 요청에 문제가 있어 서버가 요청을 처리할 수 없는 경우입니다.

프론트엔드 개발에서 특히 자주 확인하게 되는 구간입니다.

| 코드 | 상태 | 의미 |
| --- | --- | --- |
| `400` | Bad Request | 잘못된 요청 형식이나 문법으로 인해 서버가 요청을 이해할 수 없음 |
| `401` | Unauthorized | 인증이 필요한 요청이지만 인증 정보가 없거나 유효하지 않음 |
| `403` | Forbidden | 인증은 되었지만, 해당 리소스에 접근할 권한이 없음 |
| `404` | Not Found | 요청한 리소스를 찾을 수 없음 |
| `405` | Method Not Allowed | 해당 URI에서 허용되지 않는 HTTP 메서드를 사용함 |
| `409` | Conflict | 현재 서버 상태와 충돌이 발생함. 예: 중복 데이터 생성 요청 |

---

## 🔴 5xx — 서버 오류

클라이언트의 요청은 유효하지만, 서버 내부 문제로 인해 요청을 처리하지 못한 경우입니다.

주로 백엔드 로직, 서버 설정, 외부 시스템 연동 문제 등에서 발생합니다.

| 코드 | 상태 | 의미 |
| --- | --- | --- |
| `500` | Internal Server Error | 서버 내부에서 예기치 못한 오류가 발생함 |
| `502` | Bad Gateway | 게이트웨이 또는 프록시 서버가 상위 서버로부터 잘못된 응답을 받음 |
| `503` | Service Unavailable | 서버 점검 중이거나 과부하 상태여서 현재 요청을 처리할 수 없음 |
| `504` | Gateway Timeout | 서버 간 통신 과정에서 응답 시간이 초과됨 |

---

## 개발할 때 자주 보는 핵심 코드

실무에서 특히 자주 접하는 상태 코드는 아래와 같습니다.

- **200 OK**: 요청 성공
- **201 Created**: 데이터 생성 성공
- **204 No Content**: 요청 성공, 반환 데이터 없음
- **400 Bad Request**: 잘못된 요청
- **401 Unauthorized**: 인증 필요
- **403 Forbidden**: 권한 없음
- **404 Not Found**: 리소스를 찾을 수 없음
- **409 Conflict**: 중복 또는 상태 충돌
- **500 Internal Server Error**: 서버 내부 오류
- **503 Service Unavailable**: 서버 점검 또는 과부하

---

## 한눈에 보는 정리

- **1xx**: 요청 진행 중
- **2xx**: 요청 성공
- **3xx**: 추가 동작 필요
- **4xx**: 클라이언트 요청 문제
- **5xx**: 서버 문제
