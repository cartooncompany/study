# 객체 지향 프로그래밍(OOP)과 Java 핵심 정리

객체 지향 프로그래밍(Object-Oriented Programming, OOP)은

현실 세계의 사물이나 개념을 **객체(Object)** 로 모델링하고,

이 객체들이 서로 상호작용하도록 프로그램을 설계하는 방식입니다.

절차 중심으로 코드를 나누는 대신,

**데이터와 기능을 하나의 단위로 묶어 관리**한다는 점이 핵심입니다.

---

## 1. 클래스(Class)와 객체(Object)

### 클래스(Class)

클래스는 객체를 만들기 위한 **설계도**입니다.

객체가 가져야 할 **상태(필드)** 와 **행동(메서드)** 을 정의합니다.

- **필드(Field)**: 객체의 데이터, 상태
- **메서드(Method)**: 객체의 동작, 기능

### 객체(Object)

객체는 클래스를 바탕으로 실제 메모리에 생성된 **실체(인스턴스)** 입니다.

즉,

- 클래스 = 설계도
- 객체 = 설계도로 만든 실제 결과물

### 예제

```
// 클래스(설계도)
class Car {
    String color; // 상태(필드)

    void drive() { // 행동(메서드)
        System.out.println("자동차가 달립니다.");
    }
}

public class Main {
    public static void main(String[] args) {
        // 객체 생성
        Car myCar = new Car();

        myCar.color = "Red";
        myCar.drive();
    }
}
```

### 이해 포인트

위 코드에서 `Car`는 자동차의 공통 구조를 정의한 클래스이고,

`myCar`는 실제로 생성된 객체입니다.

즉, `myCar`는 다음을 가집니다.

- `color`라는 상태
- `drive()`라는 행동

---

## 2. 왜 OOP를 사용하는가?

객체 지향 프로그래밍은 단순한 문법이 아니라,

**유지보수성과 확장성을 높이기 위한 설계 방식**입니다.

### OOP의 주요 장점

- 코드 재사용성 증가
- 유지보수 용이
- 역할과 책임 분리
- 복잡한 시스템을 구조적으로 관리 가능
- 변경에 유연한 코드 작성 가능

예를 들어 프로그램이 커질수록 코드가 복잡해지는데,

OOP는 이를 객체 단위로 나누어 관리하게 해줍니다.

# 3. OOP의 4대 핵심 특징

---

## 3-1. 캡슐화(Encapsulation)

캡슐화는 **데이터와 그 데이터를 처리하는 메서드를 하나로 묶고**,

외부에서 내부 데이터에 직접 접근하지 못하도록 제한하는 개념입니다.

즉, 객체의 내부 구현을 숨기고

필요한 방식으로만 접근하도록 만드는 것입니다.

### 목적

- 잘못된 값이 들어오는 것을 방지
- 객체의 상태를 안전하게 보호
- 내부 구현 변경 시 외부 영향 최소화

### Java에서의 구현 방법

Java에서는 주로 **접근 제어자**를 사용합니다.

- `private`: 외부 직접 접근 불가
- `public`: 외부에서 접근 가능
- 필요 시 getter/setter 제공

### 예제

```
class Person {
    private int age; // 외부에서 직접 접근 불가

    public void setAge(int age) {
        if (age >= 0) {
            this.age = age;
        } else {
            System.out.println("나이는 음수가 될 수 없습니다.");
        }
    }

    public int getAge() {
        return this.age;
    }
}
```

### 왜 중요한가?

만약 `age`를 public으로 열어두면 아래처럼 문제가 생길 수 있습니다.

```
person.age=-10;
```

이런 비정상적인 값이 들어가는 것을 막기 위해

setter 내부에서 검증 로직을 둘 수 있습니다.

### 핵심 정리

캡슐화는 단순히 `private`를 쓰는 것이 아니라,

**객체가 자신의 상태를 스스로 책임지고 관리하게 만드는 설계**입니다.

---

## 3-2. 상속(Inheritance)

상속은 기존 클래스의 필드와 메서드를

새로운 클래스가 물려받아 사용하는 개념입니다.

Java에서는 `extends` 키워드로 표현합니다.

### 목적

- 기존 코드 재사용
- 중복 제거
- 클래스 간 계층 구조 표현

### 예제

```
// 부모 클래스
class Animal {
    void eat() {
        System.out.println("먹이를 먹습니다.");
    }
}

// 자식 클래스
class Dog extends Animal {
    void bark() {
        System.out.println("멍멍!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.eat();   // 부모에게서 상속받은 메서드
        dog.bark();  // 자식 클래스의 고유 메서드
    }
}
```

### 이해 포인트

`Dog`는 `Animal`의 특징을 물려받습니다.

즉, 강아지는 동물의 한 종류이므로 자연스러운 관계입니다.

- `Animal` → 공통 기능 정의
- `Dog` → 공통 기능 + 자신의 기능 확장

### 상속의 장점

- 공통 로직을 부모 클래스에 모아 중복 제거
- 관련 클래스들을 계층적으로 표현 가능

### 주의할 점

상속은 강력하지만 무조건 많이 쓰는 것이 좋은 것은 아닙니다.

상속은 클래스 간 결합도를 높일 수 있기 때문에,

단순 코드 재사용 목적이라면 **합성(Composition)** 이 더 적절한 경우도 많습니다.

예를 들어:

- **상속**: "A는 B의 한 종류이다"
- **합성**: "A는 B를 가진다"

즉, 상속은 **is-a 관계**일 때 적합합니다.

## 3-3. 다형성(Polymorphism)

다형성은 하나의 인터페이스나 부모 타입으로

여러 객체를 동일하게 다룰 수 있는 성질입니다.

즉, **같은 이름이나 같은 타입을 사용하더라도 실제 동작은 달라질 수 있음**을 의미합니다.

Java에서는 주로 다음 두 가지로 나타납니다.

- **오버로딩(Overloading)**
- **오버라이딩(Overriding)**

### 1) 오버로딩(Overloading)

같은 이름의 메서드를

매개변수의 개수나 타입을 다르게 하여 여러 개 정의하는 방식입니다.

```
class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
}
```

### 특징

- 같은 클래스 안에서 발생
- 메서드 이름은 같고, 시그니처가 달라야 함

### 2) 오버라이딩(Overriding)

부모 클래스로부터 물려받은 메서드를

자식 클래스에서 **재정의**하는 방식입니다.

```
class Parent {
    void print() {
        System.out.println("부모입니다.");
    }
}

class Child extends Parent {
    @Override
    void print() {
        System.out.println("자식입니다.");
    }
}
```

### 특징

- 상속 관계에서 발생
- 메서드 이름, 매개변수, 반환 타입이 규칙에 맞아야 함
- `@Override`를 붙이면 실수를 줄일 수 있음

### 다형성의 진짜 핵심

다형성의 핵심은 단순 오버로딩/오버라이딩보다,

**부모 타입으로 여러 자식 객체를 다룰 수 있다**는 점입니다.

```
class Animal {
    void sound() {
        System.out.println("동물이 소리를 냅니다.");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("멍멍!");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("야옹!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.sound();
        a2.sound();
    }
}
```

### 실행 결과

```
멍멍!
야옹!
```

여기서 변수 타입은 둘 다 `Animal`이지만,

실제 객체가 `Dog`, `Cat`이기 때문에 서로 다르게 동작합니다.

### 장점

- 코드 확장에 유리
- 조건문 감소
- 유연한 설계 가능

예를 들어 새로운 동물 `Bird`가 추가되어도

기존 코드를 크게 수정하지 않고 확장할 수 있습니다.

## 3-4. 추상화(Abstraction)

추상화는 복잡한 내부 구현을 숨기고,

사용에 필요한 핵심 기능만 외부에 드러내는 개념입니다.

즉,

**"어떻게 동작하는지"보다 "무엇을 하는지"에 집중하게 만드는 것**입니다.

### 목적

- 복잡성 감소
- 공통 규격 정의
- 구현과 사용 분리

### Java에서의 구현 방법

- `abstract class`
- `interface`

### 인터페이스 예제

```
interface Playable {
    void play();
}

class AudioPlayer implements Playable {
    @Override
    public void play() {
        System.out.println("음악을 재생합니다.");
    }
}

class VideoPlayer implements Playable {
    @Override
    public void play() {
        System.out.println("영상을 재생합니다.");
    }
}
```

### 이해 포인트

`Playable`은 “재생할 수 있는 객체”의 규격을 정의합니다.

구체적으로 어떻게 재생하는지는 각 클래스가 알아서 구현합니다.

- `AudioPlayer` → 음악 재생
- `VideoPlayer` → 영상 재생

### 추상화의 장점

- 공통 인터페이스 제공
- 구현체 교체가 쉬움
- 협업 시 역할 분담이 명확해짐

예를 들어 한 팀은 `interface`만 정의하고,

다른 팀은 그 규격에 맞는 구현체를 만들 수 있습니다.

---

# 4. OOP 4대 특징의 관계

이 네 가지는 서로 독립된 개념이 아니라,

실제 객체 지향 설계에서 함께 사용됩니다.

### 예시 흐름

- **캡슐화**: 객체 내부 상태 보호
- **상속**: 공통 기능 재사용
- **다형성**: 같은 방식으로 다양한 객체 처리
- **추상화**: 핵심 기능만 드러내고 구조 단순화

즉, 좋은 OOP 설계는

단순히 문법을 사용하는 것이 아니라

이 특징들을 적절히 조합해서 **유지보수하기 쉬운 구조**를 만드는 것입니다.

# 5. 핵심 요약표

| 개념 | 의미 | 비유 | Java 핵심 요소 | 주요 효과 |
| --- | --- | --- | --- | --- |
| 캡슐화 | 데이터와 메서드를 하나로 묶고 외부 접근 제한 | 알약 | `private`, getter/setter | 데이터 보호, 무결성 유지 |
| 상속 | 부모 클래스의 기능을 자식이 물려받음 | 유산 상속 | `extends` | 코드 재사용, 중복 제거 |
| 다형성 | 같은 타입으로 여러 객체를 다르게 다룸 | 만능 리모컨 | `@Override`, 오버로딩 | 유연성 증가, 확장성 향상 |
| 추상화 | 핵심 기능만 드러내고 내부 구현 숨김 | 자동차 운전대 | `interface`, `abstract` | 복잡성 감소, 설계 표준화 |

# 6. 한 번에 이해하는 핵심 문장

- **클래스**는 객체를 만들기 위한 설계도이다.
- **객체**는 클래스로부터 생성된 실제 인스턴스이다.
- **캡슐화**는 객체의 상태를 안전하게 보호한다.
- **상속**은 공통 기능을 재사용하게 해준다.
- **다형성**은 같은 타입으로 다양한 객체를 유연하게 다루게 해준다.
- **추상화**는 복잡한 구현을 감추고 핵심 기능에 집중하게 해준다.

---

# 7. 결론

Java에서의 객체 지향 프로그래밍은

단순히 클래스와 객체를 만드는 문법 학습이 아니라,

**프로그램을 더 구조적이고 유연하게 설계하기 위한 사고방식**입니다.

특히 OOP의 4대 특징은 서로 연결되어 있으며,

잘 활용하면 다음과 같은 코드를 만들 수 있습니다.

- 읽기 쉬운 코드
- 수정하기 쉬운 코드
- 확장하기 쉬운 코드
- 재사용 가능한 코드

즉, OOP의 본질은

**현실 세계를 닮은 구조로 프로그램을 조직하고, 변화에 강한 소프트웨어를 만드는 것**이라고 볼 수 있습니다.
