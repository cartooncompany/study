# UDP 정리

UDP(User Datagram Protocol)는 인터넷 전송 계층의 대표적인 프로토콜 중 하나로, **빠르고 단순한 전송**에 초점을 둡니다.  
UDP는 **연결 설정 없이** 데이터를 보내며, **도착 보장, 순서 보장, 재전송** 같은 기능을 자체적으로 제공하지 않습니다.  
대신 헤더가 매우 작고 처리 과정이 단순해서 **지연을 줄이는 데 유리**합니다.

---

## 1. UDP의 핵심 특징

### 1) 비연결형 (Connectionless)
TCP는 통신 전에 연결을 설정하는 절차를 거치지만, UDP는 그런 과정 없이 바로 데이터를 보냅니다.  
즉, **상대방과 연결을 맺지 않고 바로 전송**하는 방식입니다.

### 2) 비신뢰형 (Unreliable)
UDP는 데이터를 보낸 뒤 **상대가 잘 받았는지 확인하지 않습니다.**  
그래서 다음을 보장하지 않습니다.

- 도착 보장
- 순서 보장
- 중복 방지
- 자동 재전송

즉, 일부 데이터가 유실되거나, 순서가 바뀌거나, 중복 도착할 수 있습니다.

### 3) 낮은 오버헤드와 빠른 처리
UDP는 기능이 단순하기 때문에 **헤더 크기가 8바이트**로 매우 작습니다.  
TCP의 기본 헤더가 20바이트인 것과 비교하면 훨씬 가볍습니다.

- **UDP 헤더:** 8바이트
- **TCP 기본 헤더:** 20바이트

이 덕분에 네트워크 부담이 적고, 일반적으로 **지연이 낮은 통신**에 유리합니다.

### 4) 데이터그램 단위 전송
UDP는 데이터를 **데이터그램(datagram)** 단위로 보냅니다.  
즉, 보낸 메시지의 경계가 유지됩니다.  
반면 TCP는 바이트 스트림 기반이라, 메시지 경계를 애플리케이션이 따로 관리해야 합니다.

### 5) 체크섬 지원
UDP는 완전히 아무 검사도 하지 않는 것은 아닙니다.  
헤더에 **체크섬(checksum)** 필드가 있어 데이터 손상을 감지할 수 있습니다.  
다만, 손상을 감지하더라도 **재전송은 자동으로 하지 않습니다.**

---

## 2. TCP vs UDP 한눈에 비교

| 구분 | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
|---|---|---|
| 목표 | 데이터의 신뢰성과 정확한 전송 | 데이터의 빠른 전송 |
| 연결 방식 | 연결 지향형 | 비연결형 |
| 전송 보장 | 도착/순서/재전송 보장 | 보장하지 않음 |
| 전송 순서 | 보낸 순서대로 도착 | 순서가 바뀔 수 있음 |
| 재전송 | 있음 | 없음 |
| 흐름 제어 / 혼잡 제어 | 있음 | 없음 |
| 데이터 단위 | 바이트 스트림 | 데이터그램 |
| 헤더 크기 | 기본 20바이트 | 8바이트 |
| 대표 용도 | 웹, 파일 전송, 이메일 | 실시간 통신, 게임, DNS 등 |
| 비유 | 꼼꼼한 등기 택배 | 빠른 방송 송출 |

## 3. UDP는 언제 사용될까?

UDP는 **약간의 손실보다 빠른 반응이 더 중요한 상황**에서 주로 사용됩니다.

### 1) 실시간 음성/영상 통신
영상 통화나 음성 채팅에서는 약간의 깨짐이 있더라도  
**지금 시점의 데이터를 빠르게 받는 것**이 더 중요합니다.

예:
- 인터넷 전화(VoIP)
- 화상 회의
- 음성 채팅

### 2) 온라인 게임
FPS, 격투 게임처럼 **반응 속도(핑)** 가 중요한 게임에서는 UDP가 자주 사용됩니다.  
총을 쏘거나 캐릭터를 움직일 때 지연이 적어야 하기 때문입니다.

### 3) DNS
DNS는 도메인 이름을 IP 주소로 바꿔주는 시스템입니다.  
DNS 요청과 응답은 보통 데이터 크기가 작고, 빠르게 처리되어야 하므로 UDP가 많이 사용됩니다.

예:
- `google.com` → IP 주소 조회

### 4) 실시간 스트리밍 및 미디어 전송
실시간성이 중요한 미디어 전송에서는 UDP가 유리할 수 있습니다.  
다만 현대 웹 서비스에서는 TCP 기반 전송이나, UDP 위에서 동작하는 QUIC 같은 방식도 함께 많이 사용됩니다.

## 4. UDP의 장점과 단점

### 장점
- 연결 설정이 없어 시작이 빠름
- 헤더가 작아 오버헤드가 적음
- 지연이 낮아 실시간 서비스에 유리
- 데이터그램 단위로 메시지 경계를 유지함
- 구현이 단순함

### 단점
- 도착 보장 없음
- 순서 보장 없음
- 재전송 없음
- 흐름 제어 / 혼잡 제어 없음
- 중요한 데이터 전송에는 부적합할 수 있음

---

## 5. 핵심 요약

UDP는 **빠르고 가벼운 비연결형 전송 프로토콜**입니다.  
대신 **신뢰성은 낮기 때문에**, 데이터가 조금 손실되어도 괜찮고 **실시간 반응이 중요한 서비스**에 적합합니다.

즉:

> **UDP는 "정확성보다 실시간성"이 중요한 상황에 적합한 프로토콜이다.**
